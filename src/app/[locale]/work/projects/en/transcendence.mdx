## Introduction

Welcome to our project **Transcendence**!

Transcendence is the final project of the core curriculum at 42. The goal is to create an interactive website that allows users to play the famous game **Pong** online against other players. This group project enables us to put into practice all the skills we've acquired during our studies, covering aspects such as web development, real-time communication, microservices architecture, and application security.

### Prerequisites

To complete this project, several skills and knowledge areas are required:

- **Front-End Development**: Proficiency in HTML, CSS, and JavaScript to build a responsive and attractive user interface.
- **Back-End Framework**: Experience with Django to handle server-side logic, APIs, and WebSockets.
- **Databases**: Understanding of database management systems to store user information and game data.
- **WebSockets**: Knowledge of real-time communication protocols to enable instant interactions between players.
- **DevOps**: Familiarity with tools like Docker for containerization, and the ELK Stack for monitoring and log management.
- **Cybersecurity**: Application of best practices in cybersecurity to protect sensitive data and secure services.

This project is an exciting challenge that allows us to collaborate effectively while solidifying our technical skills.

## Our Project

To begin this work, we divided the tasks as follows:
1. Geoffrey handled Django, the database, WebSockets, and game physics (and other things, but that's not important).
1. Luke took care of the Front-End, the Django connections, and game graphics (a task I absolutely did not want to manage).
1. Finally, since I was ahead of the other two, I focused on the infrastructure, including microservices architecture, DevOps, security between services, etc.

### Our Modules

Here is a brief list of the modules we implemented:

#### Major:
- Django
- User management
- Remote players
- Live chat
- Application firewall + Vault
- Elastic Stack
- Microservices
- 3D ThreeJs for game rendering
- Server-side game rendering

#### Minor:
- Database
- Statistics
- GDPR
- Performance monitoring with Prometheus and Grafana
